🔌 EV Go-Kart CAN Telemetry System

Formula Kart Racing Championship Season 06 | March 2025 - March 2026

ESP32 + MCP2515 Setup

Real-time Speed Display

Go-Kart Integration

📋 Project Overview

Developed a complete CAN bus telemetry system for an electric go-kart competing in Formula Kart Racing Championship Season 06. The system reads motor data (RPM, temperature, voltage) from the VESC controller via CAN protocol and displays real-time metrics on an LCD screen.

🔧 Technical Details

  • Microcontroller: ESP32 with MCP2515 CAN controller
  • CAN Protocol: 250 kbps, extended frames (29-bit IDs)
  • Display: 16x2 I2C LCD
  • Motor Controller: VESC with custom DBC file
  • Calculations: Speed from motor RPM using wheel circumference (27.94cm diameter)

📊 Results & Proof

Working Prototype: Successfully tested on actual go-kart during practice runs

Team Recognition: Contributed to telemetry subsystem for Season 06

Code Repository: 9 commits, multiple working .ino files

Documentation: Complete VESC DBC file and CAN ID mapping

🌀 Chaos-Driven Time-Varying Convolutional Codec

Research Project | FPGA Implementation | Verilog HDL

📋 Project Overview

Design and hardware implementation of a chaos-driven time-varying convolutional encoder with constraint length K=7. Unlike conventional fixed-generator encoders, this system dynamically varies generator polynomials using a synchronized chaotic sequence, introducing lightweight physical layer security while maintaining error correction capability.

🔧 Technical Details

  • Encoding: Rate 1/2 convolutional encoder with K=7
  • Polynomial Variation: Dynamic selection using logistic map chaos generator
  • Decoding: 64-state Viterbi decoder with synchronized chaos
  • Implementation: Synthesizable Verilog HDL for FPGA
  • Tools: Vivado, ModelSim, Xilinx FPGA

📊 Expected Outcomes & Proof

Simulation Results: BER comparison with fixed NASA (133,171) code

Resource Usage: LUT count, flip-flops, max clock frequency

Synchronization: Verified chaos generator synchronization for correct decoding

Technical Paper: Complete proposal with block diagrams and architecture

📟 Digital Voltmeter (ADC0808 + 8051)

Embedded Systems Project | Microcontroller Programming

📋 Project Overview

3-channel digital voltmeter using 8051 microcontroller and ADC0808. Converts analog voltage (0-5V) from potentiometers to digital display on a 16×2 LCD. Achieved 19.5mV resolution with less than 2% error across all channels.

🔧 Technical Details

  • Microcontroller: AT89S52 (8051 architecture)
  • ADC: ADC0808 with 8-channel multiplexer
  • Display: 16×2 HD44780 LCD (4-bit mode)
  • Programming: Embedded C in Keil μVision
  • Simulation: Proteus Design Suite

📊 Results & Proof

Accuracy: Average error < 2% across test voltages (0.5V - 4.5V)

Resolution: 19.5mV per step (5V/255)

Working Simulation: Successfully simulated in Proteus with real-time updates

🔐 FPGA‑Based Crypto Accelerator (AES‑128 + SHA‑256)

🚧 WORK IN PROGRESS
Hardware Security Module | Tamper‑Response | ESP32 Bridge

📋 Project Overview

Designing a dedicated hardware accelerator for AES‑128 encryption/decryption and SHA‑256 hashing, implemented in Verilog on an FPGA. The system includes a protected key storage register that locks the root key after configuration – it cannot be read out, only used internally. A physical tamper button (or sensor) instantly zeros the key, demonstrating active tamper response. Communication is handled by an ESP32 acting as a USB‑to‑SPI bridge, allowing a Python script on a PC to send encryption or HMAC commands. The final system proves hardware key separation: even if the host is compromised, the secret key stays safe inside the FPGA.

🔧 Planned Features

  • AES‑128: ECB and CBC modes, fully pipelined for high throughput
  • SHA‑256: HMAC support for authentication
  • Key Storage: Internal register with write‑only access (locked after programming)
  • Tamper Detection: Physical button triggers key zeroization and erases sensitive state
  • Host Interface: ESP32 acts as SPI bridge – commands from Python over USB
  • FPGA Target: Xilinx Artix‑7 or similar (Vivado development)

📊 Current Status

In Progress: AES‑128 core designed and simulated; SHA‑256 module under implementation

Next Steps: Integrate key storage with write‑once logic, implement tamper circuit on FPGA board, write ESP32 firmware and Python host script

Documentation: Design notes and simulation results will be added as development continues
